Modificar tablas

Modificar tablas aplica estilos de visualización y formato a tablas dinámicas. Puede
  • controlar el aspecto de casillas de datos individuales y etiquetas de filas y columnas.
  • establecer el ancho de columnas específicas u ocultarlas.
  • pivotar las dimensiones de tabla.
Puede seleccionar tipos específicos de tablas en el visor para decidir a cuáles aplicar estas modificaciones y el ámbito.

•Las tablas ligeras de la versión 19 no se ven afectadas por este comando, puesto que no se pueden editar. Está limitación no se aplica a las versiones posteriores.

La especificación de selección se compone de los siguientes elementos:

  • Los tipos de tablas que van a procesarse. Los tipos son los subtipos de tabla SGR.
  • La dimensión: filas o columnas (el valor predeterminado es columnas).
  • Las filas o columnas específicas, incluidas todas.
  • El ámbito de la aplicación: el comando precedente o todo el resultado (el valor predeterminado es el resultado precedente).
  • Criterios de selección opcionales específicos del valor de manera que el valor de la casilla debe ser superior a 2.
  • Selección opcional de celdas con marcadores de significación
Las especificaciones de la acción son:
  • Ocultar filas o columnas especificadas.
  • Establecer anchos para columnas y/o anchos de las etiquetas de fila.
  • Establecer el estilo de fuente como normal, negrita, cursiva o negrita cursiva.
  • Establecer el color de fuente.
  • Establecer el color de fondo.
  • Aplicar acciones personalizadas a las casillas especificadas.
Las especificaciones de selección se realizan en el cuadro de diálogo principal y en el cuadro de diálogo Significación. Las especificaciones de acción se realizan en el subcuadro de diálogo Acciones. Si no especifica ninguna acción o si los criterios de selección no encuentran ninguna tabla coincidente, el comando no hará nada.

Selección de las tablas, dimensión y filas o columnas

Introduzca el subtipo de tabla SGR en el campo Subtipo de tabla. EL subtipo se puede encontrar pulsando con el botón derecho en los titulares en una instancia de la tabla y seleccionando Copiar subtipo OMS o en la lista en Utilidades>Identificadores OMS. Introduzca * para seleccionar todos los subtipos.

Seleccione Columnas o Filas para determinar la dimensión de la tabla que se va a procesar. Los anchos sólo se pueden establecer para la dimensión de columna, pero los anchos de etiqueta de fila pueden establecerse independientemente de la dimensión.

En Columnas o filas seleccionadas introduzca una lista separada por espacios en blanco de las filas o columnas específicas que se deben procesar o introduzca "<<TODO>>" para procesar la dimensión entera (incluya las comillas y escriba TODO en mayúsculas). Las filas o columnas pueden especificarse por posición numérica o por el texto de etiqueta entre comillas.

Para Nivel para coincidencia de texto, la primera fila o columna (la más externa) es 0. Utilice números negativos para contar hacia atrás desde el final: -1 es la última fila o columna (las más interna). Para la coincidencia de texto de etiqueta, incluya el texto entre comillas. Las mayúsculas y minúsculas y los espacios de la etiqueta deben coincidir exactamente. Las especificaciones numéricas y de texto se pueden combinar. Por ejemplo,
2 3 "Superior"
selecciona las filas o columnas 2, 3 y una con la etiqueta "Superior". Si ningún elemento coincide con la especificación, ésta se ignorará. Las filas o columnas se cuentan en el nivel más interno de la tabla.

Si se marca El texto anterior es un patrón, cada campo de texto anterior (excepto <<TODO>>) se trata como una expresión regular. Por ejemplo, "Sig\." coincidiría con cualquier texto de etiqueta que contenga Sig. . De lo contrario, la etiqueta coincidirá literalmente con el texto especificado. Como expresión regular, coincidiría con Sig. asintótica o Sig. bilateral pero no con Sig asintótica (sin punto). Como texto normal, ninguna de las opciones coincidiría. Se puede utilizar cualquier expresión regular de Python. Los patrones distinguen entre mayúsculas y minúsculas y tienen en cuenta la configuración regional.

Los elementos que parecen números siempre se tratan como posiciones independientemente de la configuración de casilla de verificación.

Una explicación completa de las expresiones regulares está más allá del ámbito de esta ayuda, pero una característica que merece la pena destacar aquí es la capacidad de crear una expresión negativa. Supongamos, por ejemplo, que desea seleccionar todas las filas o columnas excepto las que empiecen por "Total". Puede hacerlo con
"^(?!Total)" El signo de intercalación (^) inicia la coincidencia al principio de la etiqueta. El resto sólo es verdadero si el texto siguiente no es Total.

Las expresiones regulares no se pueden utilizar con especificaciones de ancho de columnas de datos.

Cuando se busca la coincidencia de texto, se utiliza de forma predeterminada el texto del nivel más interno (-1). Introduzca más números negativos para salir (por arriba por la izquierda) o números positivos para contar de fuera hacia dentro. Tenga en cuenta que algunas tablas tienen niveles invisibles en medio lo que se ve, de manera qué es posible que se necesite un poco de experimentación para obtener el nivel correcto. El nivel 1 suele ser el nivel visible más exterior.

Marque Imprimir etiquetas para ver todas las etiquetas de la dimensión seleccionada. Como muchas tablas tienen etiquetas ocultas, utilice esta función para ver toda la estructura.

De forma predeterminada, se procesan todas las tablas de los tipos especificados en el resultado del procedimiento inmediatamente anterior. Seleccione Todos los resultados para procesar todos los procedimientos ejecutados anteriormente.

Subcuadro de diálogo Significación

Este subcuadro de diálogo delimita la selección realizada en el diálogo principal a las celdas que cumplen los criterios de significación aquí especificados. En la mayoría de los casos, los criterios de significación se utilizarían con la especificación de selección "<<TODO>>" en el cuadro de diálogo principal, pero si hay un subconjunto seleccionado aquí, las especificaciones de significación se aplican dentro de dicho subconjunto.

• Este subcuadro de diálogo se aplica solo la versión 24 o posterior de Statistics y a la salida de las tablas personalizadas (CTABLES) dónde los marcadores de significación se muestran en la tabla principal y son del tipo simple (no APA).

Selección de celdas significativas: Seleccione Todas las celdas con marcador de significación para seleccionar todas las celdas que tengan algún marcador de significación. Los marcadores de significación son códigos de letras especiales que aparecen en la celda

Cómo alternativa, se pueden especificar marcadores de significación específicos que se deberán seleccionar introduciendo un patrón de significación. El patrón puede ser una simple lista de los marcadores de las columnas. Por ejemplo, "AB" solo seleccionaría celdas en las que las columnas con las letras A o B se indican como significativamente diferentes. Los códigos de letras se muestran en las cabeceras de columna en que los resultados de significación se fusionan en lugar de mostrarse en una tabla distinta. Puede usar esta opción cuando desee centrarse en unas pocas categorías específicas.

Se producen subtablas a través de las columnas cuando la dimensión de la columna tiene una especificación anidada, como por ejemplo una categoría de trabajo dentro de un género (género > categoría_trabajo) o cuando hay múltiples variables categóricas apiladas en dicha dimensión (categoría_trabajo + género). Puede especificar adicionalmente a qué tablas se aplica la selección indicando los números de subtabla después de cada código de letra. Por ejemplo, A01B2 indica que solo las celdas significativas de A se deben seleccionar en las subtablas 0 y 1 y solo los códigos significativos de B se deben seleccionar en la subtabla 2. Puede identificar las subtablas en el resultado porque la letra de la columna correspondiente a la significación se reinicia a partir de (A) a través de la tabla para cada subtabla.

En una tabla anidada, la dimensión más interna se repite, de manera que probablemente querrá que la selección se aplique entre todas las subtablas anidadas. No obstante, si hay variables apiladas entre las columnas, el significado de las letras de significación será diferente en cada subtabla y es posible que desee que la selección se aplique solo a subtablas específicas.

• La numeración de las subtablas empieza a partir de 0. Se pueden referenciar hasta 10 subtablas utilizando esta notación.

Nivel de significación: se pueden mostrar uno o dos niveles de significación en una tabla. Si hay dos, los valores más significantes se indican en mayúsculas y los menos significantes en minúsculas. Al realizar la selección, puede especificar que ambos niveles se seleccionan o que solo se selecciona uno o el otro. Si solo hay un nivel en la tabla, seleccione Ambos o Superior.

Subcuadro de diálogo Acciones

Marque Ocultar las filas o columnas seleccionadas para que estos elementos sean invisibles. Esta opción no se puede combinar con ninguna otra acción.

Establecimiento de anchos de columna

En el campo Ancho de columna, introduzca un número en puntos que deba aplicarse a todas las columnas seleccionadas o una lista de números, tantos valores como columnas seleccionadas, para especificar anchos individuales. Un punto es 1/72 pulgadas. (Sugerencia: Para ver el ancho actual de una columna, active la tabla con una doble pulsación, pulse en el borde de una columna y arrástrelo un poco).

Los anchos de etiqueta de fila pueden controlarse introduciendo posiciones de etiqueta de fila con la misma notación que para seleccionar filas o columnas y un único ancho o un conjunto de anchos.

Aplicación de estilos

Se pueden aplicar estilos a las casillas de datos de una tabla, las etiquetas de la dimensión seleccionada o a ambas. Seleccione Casillas de datos con la condición (en el cuadro de diálogo principal) para procesar sólo las casillas de datos que cumplan la condición.

La condición es una expresión verdadero/falso en la que el valor de la celda se representa como x. Por ejemplo,

x < .05
seleccionaría únicamente celdas de datos cuyo valor sea inferior a .05. La expresión se interpreta como código Python. Se pueden utilizar los operadores lógicos y de comparación ordinarios. abs(x) es el valor absoluto de la celda.

i sustituye al número de filas o columnas en la dimensión opuesta a la dimensión SELECT. Por ejemplo
i % 2 == 1
(% es el operador mod de Python) seleccionaría las columnas con numeración impar si la dimensión seleccionada es filas.

ii sustituye a la fila o columna de la dimensión SELECT. Especificación
ii % 2 == 1
Utilizarse para aplicar un estilo a las filas con numeración impar si la dimensión SELECT es filas.

Utilice siempre un punto (.) al especificar un valor decimal aunque en su configuración regional la coma sea la separación decimal habitual.

Seleccione un estilo de texto, color y color de fondo utilizando los controles adecuados. Los colores se representan como tres números: uno para el componente rojo, uno para el verde y uno para el azul. Los valores deben estar entre 0, ninguno, y 255, máximo. Por ejemplo, 0 0 0 es negro, 255 255 255 es blanco y 255 0 0 es rojo. El amarillo se compone de partes iguales de rojo y verde.

Si tiene una función personalizada que se debe aplicar a las celdas especificadas, introdúzcala aquí como
nombremódulo.nombrefunción
Consulte el módulo de Python customstylefunctions.py para obtener información detallada y ejemplos sobre cómo escribir dichas funciones.

Las funciones personalizadas pueden tener parámetros. Para especificar uno o más parámetros, escriba los nombres y valores de parámetros e incluya la lista de parámetros entre paréntesis. Por ejemplo,
customstylefunctions.stripeOddRows2(r=20,g=20,b=200)
aplicaría la función stripeOddRows2 en el módulo customstylefunctions. Obtendrá tres parámetros, r, g y b (para rojo, verde y azul), con los valores 20, 20 y 200, respectivamente. Consulte la documentación de cualquier función personalizada en particular para conocer qué parámetros acepta. Los nombres de parámetros deben escribirse siguiendo las mayúsculas y minúsculas especificadas en la definición de función.

Si el valor del parámetro es una cadena, inclúyala entre comillas simples, por ejemplo
option='ambos'

Aspectos de tabla

Puede aplicar un aspecto de tabla a toda la tabla seleccionando Aplicar aspecto de tabla y especificando un archivo de aspecto de tabla. Los archivos de aspecto de tabla tienen la extensión stt. El aspecto de tabla se aplica a toda la tabla independientemente de la configuración del cuadro de diálogo principal.

Selección de los colores adecuados

Los mejores colores para una presentación dependen de su objetivo y las características del dispositivo en el que se visualizará. Es posible que siempre quiera enfatizar filas o columnas específicas o que quiera enfatizar valores típicos o inusuales. El mejor esquema de colores puede variar dependiendo de si se visualizará en una pantalla, impresa o proyectada. También es posible que haya un esquema de colores corporativo obligatorio. Puede obtener ayuda y coordenadas RGB en
http://www.personal.psu.edu/cab38/ColorBrewer/ColorBrewer.html .
Aunque este sitio trata sobre mapas, algunos de los consejos también pueden resultarle útiles para las tablas (o para los gráficos).

Las funciones customstylefunctions.qualitative y customstylefunctions.pastelqualitative pueden utilizarse para aplicar estos esquemas de colores a las columnas de una tabla.

Características adicionales

Este cuadro de diálogo genera la sintaxis del comando de extensión SPSSINC MODIFY TABLES. Para mostrar la ayuda de este comando, ejecute la siguiente sintaxis:

SPSSINC MODIFY TABLES /HELP.

En la sintaxis del comando puede especificar una lista de subtipos entrecomillados individualmente.

Requisitos

Este cuadro de diálogo requiere el complemento de integración de Python y el comando de extensión SPSSINC MODIFY TABLES.
